Tutorial

Limitations, requirements and basic how-it-works

It is important to understand that it may not be possible to use this extension with all sites, but if the web site you want to add follows the "standard recipe", it may be possible. Some basic understanding of HTML and CSS is probably required to pull this off.

Requirements/assumptions:

  1. Article links are <a href=".."> HTML tags.
  2. The categories of articles can be deduced from patterns in the links.
  3. Article boxes are enclosed in an outer <div> or some other HTML tag with an identifiable characteristic

Basic how-it-works:

  1. The extension will inspect all links on a page (<a> tags), and match them against the configured patterns.
  2. For each matching link, the HTML DOM will be traversed upwards via the parent containers to find the "envelope" - the <div> HTML tag or whatever represents the "article box".
  3. Once a matching parent container has been found, depending on the replacement configuration, one or more things may happen:
    1. An overlay <div> element will be inserted into the container
    2. The CSS/style of the parent container may be modified, if required

Depending on how the website in question is structured, this generic approach may or may not work.
